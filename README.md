# 智聘 AI 助手 (zhipin-ai-helper)

一个用于辅助个人求职流程的浏览器插件。目标是在不改变用户主动操作节奏的前提下，把"搜索、浏览、匹配、初始沟通"这几个高重复环节集中到同一个扩展里完成。

核心能力：

- 简历录入（`.docx` / `.txt` / `.md` 或直接粘贴文本），并支持由 AI 整理为更适合匹配的档案
- 可配置多个城市 ID 和多个搜索关键词，组合轮换执行
- 自动抓取岗位信息，调用 AI 输出匹配分数、匹配原因和沟通消息
- 对满足阈值的岗位自动进入聊天流程并发送生成的消息
- 本次运行总点击上限，降低因连续点击过多带来的风控压力
- 所有配置、缓存、日志都保存在浏览器本地

## 安装

开发者模式加载：

1. 打开 Chrome / Edge 扩展管理页
2. 开启"开发者模式"
3. 选择"加载已解压的扩展程序"
4. 选择本仓库下的 `extension/` 目录

或者构建产物后加载 `dist/unpacked/`（见下文）。

## 构建和打包

仓库内置了一个零依赖的 Node 脚本，不需要额外的前端构建工具。

```bash
# 生成可直接加载的目录：dist/unpacked/
npm run build

# 再打包成发布用 zip：dist/zhipin-ai-helper-v<版本号>.zip
npm run package
```

打包脚本会校验 `package.json` 与 `extension/manifest.json` 的版本号是否一致，每次运行前会清空 `dist/`。

## 使用步骤

启动前必须满足的条件：

- 已保存 DeepSeek API Key
- 至少配置 1 个城市 ID
- 至少配置 1 个搜索关键词
- 已填写或上传简历文本

典型流程：

1. 在插件里填写并保存 API Key
2. 配置城市 ID（支持直接粘贴职位列表 URL，自动提取 `city` 参数）
3. 配置搜索关键词，或先输入一段岗位描述让 AI 生成关键词
4. 按需设置"本次运行总点击上限"（默认 30）
5. 上传或粘贴简历，按需点击 AI 整理
6. 打开 Boss 直聘的职位列表页
7. 点击插件右上角"开始执行"

启动和停止共用同一个按钮。切换城市/关键词时直接改写列表页 URL 并强制触发一次搜索，再开始抓取。

## 配置说明

### 城市 ID

从职位列表页地址栏获取：

```
https://www.zhipin.com/web/geek/jobs?city=101250100&query=xxx
```

`city` 后面的数字即为城市 ID。插件也支持直接粘贴完整 URL，会自动提取。

### 搜索关键词

支持填写多个，每行一条：

```
AI应用工程师
AI Agent
AIGC应用开发
```

也可以先输入一段岗位描述，由 AI 生成一组适合搜索框的关键词。

### 本次运行总点击上限

单次运行内所有城市/关键词组合共享的累计点击上限，到达后自动停止。用于降低连续点击过多带来的风控风险。默认 30。

### 简历

- 支持 `.docx`、`.txt`、`.md` 上传，或直接粘贴纯文本
- 优先使用"AI 整理版"参与匹配，未整理时回退使用原始文本

## 数据与隐私

- API Key 由用户在插件内手动输入，保存在浏览器本地
- 简历文本保存在插件本地，用于岗位匹配和 AI 整理
- 岗位匹配和简历整理会把相关文本发送到外部 AI 接口（DeepSeek）
- 仓库本身不包含任何个人配置或密钥
